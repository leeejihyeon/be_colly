# 인증 실행 환경 설정 가이드

## 1. 목적
- Colly 인증 시스템을 로컬/스테이징/운영에서 실제 설정값으로 실행하기 위한 기준을 정리한다.

## 2. 기준 설정 구조
- 기준 파일: `src/main/resources/application.yaml`
- 로컬 오버라이드: `src/main/resources/application-local.yml`
- 저장소에는 실제 로컬 파일 대신 `src/main/resources/application-local.yml.example`만 둔다.
- 배포 환경은 추후 AWS Parameter Store 또는 Secrets Manager에서 같은 키 구조로 주입한다.

## 3. 로컬에서 반드시 채워야 하는 설정 키
### 공통
- `colly.auth.jwt.secret`
  - 32바이트 이상 문자열 필수
  - 미설정 시 서버는 시작 단계에서 실패한다.
- `colly.auth.jwt.issuer`
  - 기본값: `colly-api`

### 매직링크 메일
- `spring.mail.username`
- `spring.mail.password`
- `colly.auth.mail.from`
- `colly.auth.mail.magic-link-base-url`
- `colly.auth.mail.app-deep-link-base`

### Google
- `colly.auth.social.google.audience`
  - Google Web Client ID
  - 모바일 앱이 받은 `idToken`을 서버에서 검증할 때 사용한다.

### Apple
- `colly.auth.social.apple.audience`
  - iOS 앱 Bundle ID와 동일하게 맞춘다.
  - 현재 Colly 기준 값: `com.collyapp`

## 4. JWT secret 정책
- 개발/스테이징/운영 secret을 분리한다.
- 운영 secret 회전 시 기존 refresh 세션은 DB 기준으로 유지된다.
- 이미 발급된 access token은 만료 또는 재로그인 이후 새 secret 기준으로 재발급된다.

## 5. 로컬 설정 예시
1. `src/main/resources/application-local.yml.example`를 복사해 `src/main/resources/application-local.yml`를 만든다.
2. 아래 항목을 실제 값으로 채운다.

```yaml
spring:
  mail:
    username: colly.auth@gmail.com
    password: your-gmail-app-password

colly:
  auth:
    jwt:
      secret: your-32-byte-or-longer-secret
    mail:
      from: colly.auth@gmail.com
    social:
      google:
        audience: your-google-web-client-id.apps.googleusercontent.com
      apple:
        audience: com.collyapp
```


## 7. 운영 체크 포인트
- `application.yaml`에 민감값이 하드코딩되지 않았는지 확인
- `colly.auth.jwt.secret`이 32바이트 이상인지 확인
- Google Web Client ID와 `colly.auth.social.google.audience`가 일치하는지 확인
- Apple App ID / Bundle ID와 `colly.auth.social.apple.audience`가 일치하는지 확인
- Gmail SMTP를 쓸 경우 앱 비밀번호를 사용했는지 확인
- `spring.mail.username` / `colly.auth.mail.from` 주소가 실제 발신 주소와 일치하는지 확인
- 운영 환경은 로컬 파일이 아니라 외부 비밀 저장소 또는 배포 주입값으로 관리할 것을 전제로 한다
