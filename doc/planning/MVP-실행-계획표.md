# MVP 실행 계획표

## 1. 목표
- 시드니 단일 도시 기준으로 `커뮤니티 + 숙소 겹침 + 매직링크 로그인` MVP를 빠르게 출시한다.

## 2. 마일스톤
| 단계 | 기간(주) | 목표 | 산출물 |
|---|---|---|---|
| M1 | 1-2주 | 인증/회원 기초 완성 | 매직링크 로그인 API, 회원 기본 프로필 |
| M2 | 3-4주 | 커뮤니티 핵심 완성 | 게시글(모임/자유형), 참여 신청/승인, 피드 필터 |
| M3 | 5주 | 신고/제재/운영기능 | 신고 API, 3회 누적 제재 로직, 제재 상태 조회 |
| M4 | 6주 | 숙소 매칭 완성 | 숙소/체류 일정 등록, 기간 겹침 사용자 조회 |
| M5 | 7주 | 체험 탭 수익화 실험 | 체험 목록/리드 API, 제휴 링크 추적 |
| M6 | 8주 | 안정화/런칭 준비 | 통합 테스트, 문서 정리, 운영 체크리스트 |

## 3. 기능별 작업 항목
### 3.1 인증(Auth)
- 이메일 입력 기반 매직링크 발급 API
- 매직링크 검증/로그인 처리 API
- 토큰 발급(Access/Refresh) 및 만료 정책
- 재요청 쿨다운/남용 방지(이메일·IP 제한)

### 3.2 커뮤니티(Community)
- `PostType` 2종(`GATHERING`, `FREE_FEED`) 정책 검증
- 모임글 필수 필드 강제, 자유형글 모집 필드 금지
- 피드 조회 필터(`ALL/GATHERING/FREE_FEED`)
- 모임 참여/승인/거절 플로우

### 3.3 신고/제재(Moderation)
- 사용자 신고 등록 API
- 신고 누적 카운트 집계
- 3회 누적 시 7일 커뮤니티 활동 제한 자동 적용
- 제한 상태 조회 API

### 3.4 숙소 매칭(Accommodation)
- 숙소 마스터 데이터(수기 관리) CRUD
- 내 체류 일정 복수 등록/수정/삭제
- 동일 숙소·기간 겹침 사용자 조회
- 노출 정보 최소화(닉네임, 날짜)

### 3.5 체험(Experience)
- 체험 상품 노출 API
- 참여 의사/문의 리드 수집 API
- 클릭/전환 지표 수집

## 4. 품질/검증 기준
- 정책 검증 테스트
  - 자유형글에 모임 필드 입력 시 실패
  - 모임글 필수값 누락 시 실패
  - 모임 참여 API는 모임글에서만 성공
- 보안/운영 테스트
  - 매직링크 만료/재사용 방지
  - 신고 3회 누적 시 정확히 7일 제한
- 회귀 테스트
  - 사용자/주문 기존 기능 동작 유지

## 5. 릴리즈 체크리스트
- `doc/api` 도메인별 명세 최신화
- `doc/ddl` 테이블 SQL 최신화
- `doc/TODO-작업-목록.md` 미완료 항목 점검
- 에러 코드/응답 포맷 일관성 점검
