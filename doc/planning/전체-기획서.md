# 전체 기획서

## 1. 서비스 목적
- `colly`는 여행지 체류자(여행자/워홀러/배낭여행자)가 도시 단위 커뮤니티에서 사람을 만나고 일정을 함께할 수 있도록 돕는 앱이다.
- 핵심 가치는 `도시 커뮤니티 연결`, `숙소 체류 기간 겹침 기반 매칭`, `현지 체험 참여 연결`이다.

## 2. MVP 범위
- 출시 국가는 호주, 초기 도시는 `시드니` 1개로 고정한다.
- 탭 구성은 3개로 시작한다.
  - 커뮤니티 탭
  - 체험 탭
  - 마이페이지 탭
- 숙소 데이터는 초기 단계에서 외부 API 연동 없이 수기 등록/관리한다.

## 3. 핵심 기능 정의
- 커뮤니티 탭
  - 게시글 타입 2종
    - `GATHERING`: 모임 모집형 글(목적지/만남장소/시간/최대인원/참여방식 필수)
    - `FREE_FEED`: 인스타 감성의 자유형 게시판 글(사진/위치/텍스트 중심)
  - 모임 참여 방식
    - 글 작성 시 `APPROVAL(승인형)` / `FREE(자유참여형)` 선택
  - 피드 필터
    - `ALL / GATHERING / FREE_FEED`
- 신고/제재
  - 신고 누적 3회 시 7일간 커뮤니티 활동 제한(글쓰기/모임생성/모임참여 제한)
- 숙소 매칭
  - 사용자 숙소 체류 기간(체크인/체크아웃) 등록
  - 같은 숙소 + 겹치는 기간 사용자를 노출
  - 공개 정보는 `닉네임 + 겹치는 날짜`로 제한
- 체험 탭(수익화 MVP)
  - 제휴 체험 콘텐츠 노출
  - 참여/문의 리드 수집 후 수수료 기반 수익 검증

## 4. 인증 정책
- 로그인 방식은 `이메일 매직링크`를 기본으로 사용한다.
- 아이디/비밀번호 방식은 MVP에서 제외한다.
- 매직링크는 단기 만료 + 1회 사용 원칙을 적용한다.

## 5. 운영/확장 정책
- 도시는 향후 다중 도시/다국가 확장을 전제로 설계한다.
- 비용 제약을 고려해 초기 리워드는 비금전 혜택(배지/노출 우선권/제휴쿠폰 중심)으로 운영한다.
- 기획, API, DDL, TODO 문서는 코드 변경과 함께 동기화한다.
